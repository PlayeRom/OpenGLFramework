Opis szczegółowy

Manager zarządzajacy obiektami 3D, wczytanymi z autorskiego formatu pliku 3DObj.

Zadaniem tej klasy jest zapewnienie wczytania plików 3DObj oraz umożliwienie rysowania trójwymiarowych obiektów zawartych w tych plikach.

GLvoid C3DObjManager::Initialization (  ) 

Metoda inicjalizująca ustawienia 3D Object Manager-a.

Metoda ta jest już wywoływana w CGameControl::LoadAll3DObjFiles i musi być wywołana na samym końcu, przed wczytywaniem plików 3DObj.

GLint C3DObjManager::Create3DObject ( LPCTSTR  lpFileName,
GLint  iTextureFirst,
GLint  iTextureCount = -1,
GLint  iTextureEmbossBump = -1 
)

Metoda wczytująca plik 3DObj.

Metodę tą należy wywoływać wewnątrz CGameControl::LoadAll3DObjFiles tyle razy ile chcemy wczytać obiektów 3DObj.

Parametry:
[in] lpFileName Nazwa pliku z rozszerzeniem 3DObj, który chcemy wczytać.
[in] iTextureFirst Indeks tekstury jaka zostanie nałożona na obiekt. Indeksy liczone są od 0 w kolejności wczytywania tekstur w metodzie CGameControl::LoadAllTextures. Jeżeli zamierzamy wykorzystać multiteksturing, parametr ten stanowi indeks pierwszej tekstury jaka zostanie nałożona na obiekt.
[in] iTextureCount Tylko dla multiteksturingu – ilość kolejnych tekstur po iTextureFirst, jakie zostaną nałożone na obiekt. Aby miało sens, należy podać co najmniej wartość 2. Jeżeli podamy wartość mniejszą od 2, multiteksturing nie zostanie użyty. Domyślna wartość dla iTextureCount wynosi -1, czyli bez multiteksturingu.
[in] iTextureEmbossBump Indeks tekstury jaka zostanie nałożona na obiekt w celu uzyskania efektu wybojów. Musi to być specjalnie do tego celu przygotowana tekstura, najczęściej w odcieniach szarości. Należy pamiętać, że jeżeli chcemy wykorzystać teksturę bump to parametr iTextureCount musi być równy wartości domyślnej, tj. -1. Jeżeli nie chcemy wykorzystać mapowania wybojów to należy parametr ten ustawić na wartość domyślną, czyli -1.
Zwraca:
Indeks nowo stworzonego obiektu. W przypadku wystąpienia błędu, zwracana jest wartość -1.

GLvoid C3DObjManager::Draw3DObject ( GLint  iIndex,
GLboolean  bUseEmbossBump = GL_FALSE,
GLuint  uiLight = GL_LIGHT0 
)

Metoda rysująca pojedynczy obiekt.

Za pomocą tej metody możemy narysować pojedynczy obiekt wczytany z pliku 3DObj. Jednak dla lepszej wydajności zaleca się użyć C3DObjManager::Draw3DObject_Lists.

Parametry:
[in] iIndex Indeks obiektu jaki chcemy narysować. Indeksy liczone są od 0 w kolejności wczytywania obiektów, tj. w kolejności wywoływania metody C3DObjManager::Create3DObject.
[in] bUseEmbossBump Ustawiając na GL_TRUE obiekt będzie rysowany z efektem mapowania wybojów. Aby jednak tak się stało należy uprzednio w metodzie C3DObjManager::Create3DObject podać indeks tekstury bump oraz ilość kolejnych tekstur dla multiteksturingu pozostawić w wartości domyślnej. Ustawiając na GL_FALSE (wartość domyślna) efekt mapowania wybojów nie będzie używany przy rysowaniu obiektu, nawet jeżeli przy tworzeniu obiektu podaliśmy teksturę bump.
[in] uiLight Identyfikator światła, używany jest tylko dla efektu mapowania wybojów. Należy użyć GL_LIGHTx, gdzie x to cyfra od 0 do 7.
